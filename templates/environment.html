Development and Runtime Environment

A development environment is a software setup that enables data scientists, researchers, and developers to build, test, and deploy machine learning models. This project primarily uses Colab, a cloud-based Jupyter notebook environment. We choose Colab because it is free and shareable by multiple developers. It allows developers to work with data flexibly and interactively, experiment with different models and algorithms, and collaborate with others easily. Most importantly, it has sufficient computing power to load and process our large datasets (4GB of data and over 3.67 million rows) that may not be feasible to handle on a personal computer.

The trained models are saved in Python's pickle format and deployed to Google Cloud runtime environment. We chose Google Cloud as the platform for this project because all our team members are already familiar with the platform. Using a platform that the team already knows can save time and resources that would otherwise be spent learning a new platform. In addition, Google Cloud offers a $300 and three-month free trial; after that, Google Cloud offers a free tier that allows the application to continue running with limited resources without incurring any charges.

The API and the website run on FastAPI, a modern, fast (hence the name), and highly scalable web framework for building APIs with Python 3.7 and above. In addition, it leverages Python's type annotations, which provide a self-documenting codebase and helps to eliminate common errors during development. It also uses the asynchronous programming paradigm to make it highly performant.

The FastAPI application, main.py, load the saved models from the file system and accepts incoming HTTP requests. It then invokes the models' predict() functions for results.

For all the python libraries needed for this project, please refer to the requirement.txt.

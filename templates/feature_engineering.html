Feature Engineering

Feature engineering is the process of selecting, transforming, and extracting the most relevant features or variables from raw data to improve the performance of machine learning algorithms. It is a crucial step in the machine learning pipeline because the quality of the training features directly impacts the model's accuracy and efficiency.

Feature Engineering
We take the following steps to select the features for the model:

Correlation Analysis: Check the correlation of each feature with the target variable and with other features. Suppose a feature has a low correlation with the target variable and a high correlation with other features. In that case, it might not add much unique information to the model and can be dropped.

Univariate Analysis: Perform univariate analysis on each feature to determine if it has any predictive power. Highly skewed and low variance features usually have low predictive power and can be dropped.

Domain Knowledge: Use your domain knowledge to identify irrelevant or redundant features. By interviewing our subject matter experts such as OBGYN doctors, midwives, and pediatricians, we are able to narrow down our list of features.

Other advanced features selection techniques such as Principal Component Analysis (PCA), Singular Value Decomposition (SVD), or Lasso Regression to identify the importance of the features have been considered but due to the time limitation of the project, we decided not to sue them.

Data Wrangling
Data wrangling is the process of identifying and correcting or removing errors and inconsistencies in data before it is used for machine learning modeling. Data cleaning is essential because the accuracy and performance of the model heavily depend on the quality of the data used for training. The following are some techniques used in cleaning the CDC dataset. For more details, please refer to the source code

Handling missing or NA values: instead of removing records with missing data, we attempt to fill them with statistical values such as mean or mode, or assign a new value such as "Unknown" to the NA.

Inconsistent data: The CDC dataset contains height, weight, and BMI (Body Mass Index). BMI can be derived from height and weight and to correct the inconsistency, we re-compute the BMI using height and weight.

Handling outliers: Outliers are records that deviate significantly from the norm. For example, for unknown height, CDC assigns a value of 99, and it becomes an abnormal outlier value. We correct it by assigning the statistical mean to it.
